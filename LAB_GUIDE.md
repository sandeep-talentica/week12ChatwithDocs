# Week 12: Chat with Docs (RAG) - Bootcamp Lab Guide

Welcome to the Week 12 Bootcamp Lab: Chat with Docs! In this lab, we will be exploring the implementation of a Retrieval-Augmented Generation (RAG) system using Spring AI. You will learn how to:
- Process and upload documents (PDF, DOCX, TXT) into a Vector Database.
- Perform semantic search across documents.
- Ask questions and generate contextual answers based on your uploaded documents.
- Customize prompts to tailor the Assistant's responses.

---

## 1. Prerequisites and Setup

### 1.1 Project Overview
This application provides two main functionalities:
1. **Document Upload:** Read files, intelligently split them into smaller chunks, embed them into vectors, and store them in PgVector.
2. **Chat with Docs (RAG):** Accept user queries, search the vector database for relevant chunks, and provide these chunks to an LLM alongside a prompt to generate an answer.

### 1.2 Configuration (`application.properties`)
Before running the application, make sure your configurations in `src/main/resources/application.properties` are set as needed:
```properties
# Document Processing
document.processing.chunk-size=800
document.processing.chunk-overlap=100

# RAG Settings
rag.top-k=5
rag.similarity-threshold=0.7

# OpenAI Models
spring.ai.openai.chat.options.model=gpt-4o-mini
spring.ai.openai.chat.options.temperature=0.7
spring.ai.openai.embedding.options.model=text-embedding-3-small
```
Ensure that you have your OpenAPI credentials set in your `.env` file correctly. Verify that your PgVector database is running (if using Docker Compose, check `compose.yaml`).

### 1.3 Running the Application

## 2. Lab Part 1: Document Upload

The first step in a RAG system is populating the vector database. We'll use the Document Upload API to ingest documents.

### 2.1 The Upload Endpoint
**Endpoint:** `POST /api/documents/upload`
This takes a file (PDF, TXT, DOCX) and chunks it (using `TokenTextSplitter`), generates metadata, and stores it in PgVector.

**Task:** Upload a sample text file or PDF to your endpoint.
```bash
# Example: Uploading a text file
curl -X POST http://localhost:8080/api/documents/upload \
  -F "file=@notes.txt"

# Example: Uploading a PDF file
curl -X POST http://localhost:8080/api/documents/upload \
  -F "file=@document.pdf"
```

**Understanding the Response:**
You should receive a JSON response showing successful processing.
```json
{
  "success": true,
  "filename": "document.pdf",
  "chunksCreated": 5,
  "timestamp": "2026-01-25T10:13:00.000000",
  "message": "Document processed and stored successfully",
  "errorMessage": null
}
```

## 3. Lab Part 2: Chatting with Documents (RAG)

Once documents are in the vector database, you can start querying them.

### 3.1 The Chat Endpoint
**Endpoint:** `POST /api/chat`
This endpoint takes a JSON payload with a `query` and optionally `topK` (number of similar chunks to retrieve).

**Task:** Ask questions about the document you just uploaded.
```bash
# General question
curl -X POST http://localhost:8080/api/chat \
  -H "Content-Type: application/json" \
  -d '{"query": "What are the main topics covered?", "topK": 5}'
```

**Understanding the Response:**
Notice how the response includes the `answer` generated by the LLM and the `sources` (chunks and filenames) it used as context.
```json
{
  "success": true,
  "answer": "Spring AI is a document processing system that utilizes advanced natural language processing techniques...",
  "sources": ["document.pdf", "notes.txt"],
  "chunksUsed": 3,
  "timestamp": "2026-01-25T10:13:15.012262",
  "errorMessage": null
}
```

**More Technical Queries:**
```bash
curl -X POST http://localhost:8080/api/chat \
  -H "Content-Type: application/json" \
  -d '{"query": "How does chunking work?", "topK": 3}'
```

## 4. Lab Part 3: Prompt Engineering

The system uses an external prompt template for maintainability. Modifying the `.st` file allows you to instruct the LLM on how to behave without changing Java code.

### 4.1 Customizing the Prompt Template
The current RAG template is located at: `src/main/resources/prompts/rag-prompt-template.st`

**Current Template:**
```text
You are a helpful assistant that answers questions based on the provided context.

Context:
{context}

Question: {query}

Answer the question based only on the context provided above. If the context doesn't contain enough information to answer the question, say "I don't have enough information to answer that question based on the available documents."

Answer:
```
*Note: `{context}` and `{query}` are placeholders replaced by Spring AI's `PromptTemplate` at runtime.*

### 4.2 Hands-On: Modify the Template for Structured Answers
Edit `rag-prompt-template.st` to force the LLM to return formatted responses.

**Task:** Replace the file's content with the following template:
```text
You are a helpful assistant. Answer questions using the provided context.

Context:
{context}

Question: {query}

Format your answer as:
- **Summary**: (A brief 1-2 sentence answer)
- **Details**: (Supporting information drawn from the context)
- **Sources**: (Mention which parts of the context were most relevant)

If there is insufficient information in the context, explicitly say: "Insufficient data in uploaded documents."

Answer:
```

## 5. Troubleshooting Matrix

If things aren't working as expected, refer to these common issues:

- **"Connection Refused" / Port 8080 in use:** Make sure no other web servers are running. You can change port via `server.port=8081` in `application.properties`.
- **API Errors / 500 Responses from OpenAI:** Confirm your API key in the `.env` file is correct and properly loaded by the application.
- **"No documents found" / Poor Answers:**
  1. Did you upload a document successfully?
  2. Is your PgVector container running? (run `docker ps`)
  3. Try increasing `rag.top-k` or lowering `rag.similarity-threshold` in `application.properties`.
- **Database/Docker compose error**: Note from `HELP.md` that you may need a valid `compose.yaml` with DB configuration to run the app properly! Make sure your PgVector database is correctly started.
